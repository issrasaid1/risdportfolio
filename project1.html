Responsive Redesign

RISD Digital Tools for Artists 2020

For this project, I redesigned the homepage of a local restaurant's website, Brickway on Wickenden, to improve their usability, visual design, and responsiveness.
This is the link to the website: http://www.brickwayonwickenden.com
Here is a link to the redesign.

Usability problems

I analyzed the current design of the homepage by looking at three key factors: the usability, learnability, and memorability of the design.

I first looked at the usability of the design. For one, the pause and play buttons do not change on hover or on click. Therefore, it is difficult for the user to immediately see that their action with the button has worked. There is also no indication of the active page. In addition, there is two menu tabs, (one for lunch and one for breakfast), it would be simpler for the customer to use if there was one tab with both menus included. Lastly, sometimes the gallery image doesn't load so all the user sees is a white box underneath the Gallery heading.


I next explored learnability. First, the content included in the lines of text on the left side of the page are unorganized. For one, there is no heading to this section to indicate to the user its purpose. In addition, some of the content is unrelated to one another and does not make sense given the order e.g. Call for details is written before Only the freshest ingredients. There is also little to no text hierarchy, so therefore it is difficult for the user to learn what the most important information on the page is.


I finally examined the memorability of the page. The homepage of the website is very busy and it requires users to pay close attention to the text in order to remember what information is placed where. In the same way, there are too many images on the home page (in an unorganized fashion) for the user to focus on.

Accessibility tests and findings

I next used different tools to measure the accessibility of the homepage design.

I used the screen reader VoiceOver for Mac OS. Overall, it was able to read the text and images on the website effectively with no issues. In order to measure accessibility of the website I used the screen reader Wave Web Accessibility Evaluation Tool. The main error was that there is very low contrast between text and background colors on the website. Contrast between text and background colors is necessary for all users, especially users with low vision. There is particularly little color contrast between the orange background and yellow text whenever the restaurant name is mentioned. Orange against yellow does not provide enough contrast. The screen reader also found that there was no heading structure nor page regions for the site. Lastly, there was one linked image missing alternative text and one language missing or invalid error since the gallery images were not showing up.

Wireframe

Informed by my analysis of the usability issues of this design, I mocked up wireframes using Balsamiq for three different screens: desktop, tablet, and mobile. The desktop wireframe is labeled and annotated to show the different elements of the redesign and also explain the reasoning behind these changes.

Navigation bar: I simplified the menu bar to just include one menu icon.
Headline: I simplified the page to only mention the restaurant name once and have it as the main headline in the center of the page. I made this the largest text size in order to reinforce text hierarchy principles and increase memorability of the page for the customer.
Background: I chose one image to be the background, reducing the amount of images on the page, since it was too busy before.
Text: I made the main text bigger to increase readibility.
Button: I added a button in the middle to make the navigation of the website easier. The purpose of the homepage is to get people to order from Brickway, so it is important to have an easy tool (like a button) to do so.

High-fidelity prototype

I created prototypes using Figma for three screens: desktop, tablet, and mobile. These prototypes are based on the wireframes I made. The first wireframe is annotated to provide more context as to how the webpage could be built.

To make this page responsive, I used responsive web design viewport. I also used a flexbox to hold the container for the main text on the page in order for the page to be responsive and also have all the text aligned to the middle on all screens.

Conclusion

This project allowed me to critique a website and apply those observations in redesigning it. From this process, I have learned how important certain UI principles are in regards to visual appeals and the functionality of a website.
